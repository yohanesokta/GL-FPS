FPS Engine Documentation

Guide for modular development and expansion.

1. Architecture Overview

The project is modularized to separate concerns:

  • globals.h/cpp: State management & shared variables.
  • renderer.h/cpp: All 3D and 2D drawing logic.
  • player.h/cpp: Input handling, movement, and physics.
  • loader.h/cpp: Assets loading (textures/fonts).
  • libs/d3d/: Core 3D primitive library.

2. Adding Global Variables

To add a new variable (e.g., score) that can be accessed everywhere:

1 Declare in globals.h

// Inside src/globals.h
extern int score;

2 Define in globals.cpp

// Inside src/globals.cpp
int score = 0;

3. Adding 3D Objects

To add new walls, floors, or blocks to the world:

Modify renderer.cpp

Add your code inside the drawWorld() function:

void drawWorld() {
    // Existing objects...
    d3d_draw_block(10, 0, 10, 15, 5, 15, WallTexture, 1, 1);
    
    // Custom object using a new texture
    d3d_draw_cylinder(-5, 0, -5, 0, 10, 0, CustomTexture, 1, 1, true, 32);
}

4. Adding New Functions/Modules

If you want to add a new system (e.g., an Enemy system):

1 Create Header (src/enemy.h)

#ifndef ENEMY_H
#define ENEMY_H
void updateEnemies();
void drawEnemies();
#endif

2 Create Source (src/enemy.cpp)

#include "enemy.h"
#include "globals.h"
void drawEnemies() { /* Drawing logic */ }

3 Update CMakeLists.txt

add_executable(main
    src/main.cpp
    src/enemy.cpp   # Add this line
    ...
)
